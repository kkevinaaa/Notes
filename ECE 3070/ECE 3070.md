# Section 3. 汇编 (Assembly Language)
## 1. 可编程机器
### 1.1. Stored-Program Computers
指令和和数据存储与同一mem中
基本运行逻辑：
- **Fetch**：从 memory 取 instruction
- **Decode**：解释 instruction 的含义
- **Execute**：执行 instruction
- **Update PC**：确定下一条 instruction 的地址
- 重复上述过程。
重要部件：
- **PC (Program Counter)**：保存“下一条/当前要执行的 instruction 的地址”
- **IR (Instruction Register)**：保存“当前正在执行的 instruction”
- **ALU (Arithmetic Logic Unit)**：执行算术和逻辑运算
- **Memory**：同时存 instruction 和 data
### 1.2. 指令集架构 (ISA, Instruction Set Architecture)
**ISA** 是程序员与可编程机器之间操作的规则说明。（作为interface）
规定
- 程序员可见状态（visible state）是什么
- 有哪些 instruction，以及每条 instruction 如何改变这些状态。

![[Pasted image 20260422202830.png]]

>E.g. P3070 ISA 的可见状态：
> **Memory**
     16-bit 地址
     每个地址存 16-bit word
     总空间为 64K words
   **8 个通用寄存器**
     R0 ~ R7，16-bit
   **2 个专用寄存器**
     PC: program counter，存储下一条需要执行的指令地址
     SP: stack pointer
   **4 个条件码（Condition Codes, CC）**，存储ALU运算结果
     `Z`：zero，结果为0则 Z = 1
     `N`：negative，结果为负则 N = 1
     `C`：carry，对无符号数运算，结果最高位溢出/需要借位 C = 1
     `V`：overflow，对有符号数运算，溢出 V = 1

### 1.3. Buzzword Summary
- Compiler: High-Level SW ➙ Assembly Language 
- Assembler: Assembly Language ➙ Machine Language 
- Loader: Loads Machine Language to Processor Memory

## 2. 汇编指令
ISA覆盖的instruction必须支持以下几类：
- 数据移动
- 数据运算
- 控制流
- 子程序，栈支持
### 2.1. P3070 基础指令
对例子P3070而言，分别如下：
1. Load Store Transfer
	- `LDA Rd, addr`：`Rd ← Mem[addr]`
	- `LDI Rd, imm`：`Rd ← imm`
	- `LDR Rd, Rs`：==`Rd ← Mem[Rs]`==
	区分 A: address, I: immediate value (Constant), R: register
	- `STA addr, Rs`：`Mem[addr] ← Rs`
	- `STR Rd, Rs`：`Mem[Rd] ← Rs`
	- `MOV Rd, Rs`：`Rd ← Rs`。
2. Arithmetic / Logic: 不直接访问 memory，而是在寄存器上做运算，并更新 Condition Codes
	- `ADD`, `SUB`（都是前加/减后）, `NEG`, `INCR`, `DECR`
	- `AND`, `OR`, `NOT`
	- `LSHL/LSHR` 等 shift 指令：Logic shift left/right, 移出值作为进位C

3. Branch
	- `BRA addr`：无条件跳转（goto）
	- `BRN addr`：看 `N`，若上一条指令结果为负值（N = 1），则下一条指令跳转到addr处，否则继续按顺序执行。
		if (N), PC <- addr, else PC++
	- `BRZ addr`：看 `Z`
	- `BRV addr`：看 `V`。

### 2.2. 指令格式与机器码
指令分为 opcode 和 operand 两部分
P3070 采用 **destination 在前，source 在后** 的书写规则
1. 短格式 Short Format - 1 word
	addrX opcode(10bit) ra(3bit) rb(3bit)
2. 长格式 Long Format - 2 word
	addrX opcode(10bit) ra(3bit) rb(3bit)
	addrX+1 address or data (Imm)





LDA Rd, addr
>addr X: opcode = 0000010000 ra = Rd = 111 rb = Rd = 111
>	word = 0000010000 111 111
>addr X+1: addr = hex 1234 = 0001001000110100
>hex 043F 1234

### 2.3. 寻址方式
1. Register mode: operand 直接在寄存器中，不需要额外地址字段
	e.g. `MOV Rd, Rs`、`INCR Rd`, 常为短格式
	
	**MOV Rd Rs**
	```
	opcode = 0011101000 ra = Rd = 100 rb = Rs = 101
	word = 0011101000 100 101 = 3A35
	```
	
	**INCR Rd**
	```
	opcode = 0001010000 ra = Rd = 011 rb = Rd = 011
	word = 0001010000 011 011 = 141B
	```
2. Absolute mode: operand 在 memory 中，而 地址本身直接写在 instruction 后面的第二个 word
	e.g. `LDA Rd, addr`、`STA addr, Rs`
	
	**LDA Rd, addr**
	```
	addr X: opcode = 0000010000 ra = Rd = 111 rb = Rd = 111
	word = 0000010000 111 111
	addr X+1: addr = hex 1234 = 0001001000110100
	hex 043F 1234
	```
3. Register Indirect mode: 寄存器里存放的是地址，真正的数据在该地址对应的 memory 位置 (pointer / address-of)
	e.g. `LDR Rd, Rs`、`STR Rd, Rs`
	
	**LDR Rd, Rs**
	```
	opcode = 0000100000 ra = Rd = 010 rb = Rs = 011
	word = 0000100000 010 011 = hex 0813
	```
4. Immediate mode: 操作数不是 memory 中的数据，而是 instruction 自己携带的常数
	e.g. `LDI Rd, imm`
	
	**LDI Rd, imm**
	```
	addr X: opcode=0000110000 ra = Rd = 111 rb = Rd = 111
	word = 0000110000 111 111
	addr X+1 addr = hex 1234 = 0001001000110100
	hex 0C3F 1234
	```
### 2.4. 会更改状态码的指令
1. `CMI Rd, imm` ; compare Rd with data imm, set CC
	e.g. `CMI R3, $abcd` : really calculates (R3 - $abcd) and sets the CCs
	Long Format
2. Shift Operations
	1. `LSHR Rd`、`LSHL Rs` ; logical shift right/left Rd by 1 bit
		e.g. `LSHR R2` ; logical shift right R2 by 1 bit; Insert a 0 in vacated bit; p ➙ Carry(C)
	
	2. `ASHR R2` ; arithmetic shift right R2 by 1 bit;
		Insert back the high-order bit (representing sign of 2s comp number) into the vacated bit; p ➙ Carry
		！没有`ASHL`！
	3.  `ROL R2` ; rotate R2 by 1 bit through Carry![[Pasted image 20260422222120.png]]
### 2.5. 伪操作
给编译器看，而不是处理器执行；因此不会被编译成可执行的指令
1. `.EQU` (equal)
	`ricky .EQU $56` Defines a constant $56 named "ricky"
	= \#define
2. `.DW` (direct write)
	用于定义变量初始值
3. `.ORG` (organize)
	`.ORG $a000` Stuff that follows should go in memory starting at address $a000
e.g.
```asm
	  .ORG $a000 (接下来的内容和从$a000开始放)
fred  .DW $00ff （在当前地址--$a001写入$00ff）
	  .DW $abcd 
ethel .DW $1234  （`ethel` 作为一个符号，代表的是“这块数据所在的位置”，也就是$a002）
```
