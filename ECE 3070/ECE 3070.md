## Pick Up: SystemVerilog 语法相关
### 1. 隐式线网声明（Implicit declaration of wires）
直接实例化逻辑门（`and`/`or`/`not` 等）时，它们的**输出端口**会被 SystemVerilog 自动声明为 `logic`（也就是传统的 `wire` 类型）变量。
``` systemverilog
module mux (output logic f, input logic a, b, sel); 
and #5 g1 (f1, a, nsel), // f1 是 and 门的输出 → 隐式声明为 logic 
	   g2 (f2, b, sel); // f2 是 and 门的输出 → 隐式声明为 logic 
or #5 g3 (f, f1, f2); 
not g4 (nsel, sel); // nsel 是 not 门的输出 → 隐式声明为 logic endmodule
```
如果希望关闭，可以在文件开头中加入```
```systemverilog
`default nettype none
```
### 2. $monitor
`$monitor` 是 Verilog/SystemVerilog 仿真系统任务
- 作用：**监控指定信号，只要信号发生变化，就自动打印一行信息**
- 用法：**在 initial 块里只调用 1 次**，全程自动生效
1. 辅助打印参数
	```systemverilog
	$monitor($time, ...); // 打印：0、10、20...时间点
	```
2. 格式字符串，双引号包裹
	```systemverilog
	$monitor ($time, " a=%b, b=%b, sel=%b, muxOut=%b" ,a, b, sel, muxOut);
	```
### 3. 端口连接方式
示例模块：
```systemverilog
module mux( output logic f, // 端口1：输出，名字叫f 
			input logic a, // 端口2：输入，名字叫a 
			input logic b, // 端口3：输入，名字叫b 
			input logic sel // 端口4：输入，名字叫sel ); 
// 模块内部逻辑（与/或/非门搭建） 
endmodule
```

1. 位置关联（Ordered Port Connection）
	```systemverilog
	module system; 
		logic wire_a, wire_b, select, muxOut; 
		// 位置关联实例化 
		mux DUT (muxOut, wire_a, wire_b, select); 
	endmodule
	```

2. 命名关联（Named Port Connection）
	```systemverilog
	mux DUT ( 
		.a(wire_a), // 模块端口a，连接外部信号wire_a 
		.f(muxOut), // 模块端口f，连接外部信号muxOut 
		.b(wire_b), // 模块端口b，连接外部信号wire_b 
		.sel(select) // 模块端口sel，连接外部信号select 
	);
	```
通过 `.端口名(外部信号)` 的格式，显式指定每个端口的连接对象，和顺序无关，只看名字。

3. 通配符关联（Named Port Shortcut）
	```systemverilog
	module system; // 外部信号名，和mux模块的端口名完全一致！ 
		logic a, b, sel, f; 
		// 通配符关联，自动匹配所有端口 
		mux DUT (.*); 
	endmodule
	```

## 2. Combinational Logic Components
### 2.1 多路选择器，Mux (Multiplexer)
从多个输入中选择一个，送到输出。(Select one of several inputs)
一般地，一个 $2^n:1$ Mux 有：
- $2^n$ 个 data inputs（数据输入）
- n 条 select lines（选择线 / 控制线）
- 1 个 output（输出）
1. 门级结构
	$F=S_{1}^{\prime}S_{0}^{\prime}I_{0}+S_{1}^{\prime}S_{0}I_{1}+S_{1}S_{0}^{\prime}I_{2}+S_{1}S_{0}I_{3}$
	Mux 的每一项都由“选择信号的一个 minterm”乘以“对应输入”组成。
2. 用 Mux 实现任意布尔函数
	1. $2^n:1$ Mux 实现 n-variable function
		$F(A,B,C)=m_0​+m_2​+m_6​+m_7​$
		这意味着真值表中编号为 0、2、6、7 的行输出为 1，其他行为 0。由于有 3 个变量 A,B,C，所以可以直接使用 8:1 Mux。把 A,B,C 接到选择线 ​$S_{2},S_{1},S_{0}$，然后按照真值表把每个输入 接成 0 或 1。
		**Lookup Table** 方法
	2. $2^{n−1}:1$ Mux 实现 n-variable function
		如果使用 4:1 Mux，可以把 A,B 接到选择线，把剩下的变量 C 放在数据输入端。按 AB 的四种情况分组：
		- 当 AB=00，对应 $m_0,m_1$，输出随 C 为 C′；
		- 当 AB=01，对应 $m_2,m_3$，输出也是 C′；
		- 当 AB=10，对应 $m_4,m_5$，输出为 0；
		- 当 AB=11，对应 $m_6,m_7$​，输出为 1。
		所以输出可以写为 $I_{0}=C^{\prime},\quad I_{1}=C^{\prime},\quad I_{2}=0,\quad I_{3}=1$
3. 香农展开 (Shannon Expansion) 与余因子 (Cofactor)
	1. Cofactor 的定义
		给定布尔函数：$F(x_{1},x_{2},\cdot\cdot\cdot,x_{n})$
		如果把某个变量$x_i$ 固定为 1，得到：$F_{x_{i}}=F(x_{i}=1)$ 即为 $x_i$ 的余因子。
	2. 香农展开定理
		$F=x_{i}\cdot F(x_{i}=1)+x_{i}^{\prime}\cdot F(x_{i}=0)$
		这和 2:1 Mux 完全对应。如果令选择线 $S=x_i$，那么：$$I_{0}=F(x_{i}=0),\quad I_{1}=F(x_{i}=1)$$Shannon expansion 本质上就是 “用变量 $x_i$ 作为 Mux select line，把两个 cofactor 接到 Mux 输入端”。
		
		本质：**选择几个变量作为 Mux select lines，剩下的函数作为 Mux data inputs。**
1. Mux 的 SystemVerilog 表达
	```systemverilog
	input logic [7:0] a, b;  
	output logic [7:0] f;  
	assign f = (sel) ? a : b;
	```
	或者使用always_comb
	```systemverilog
	always_comb begin
	  if (sel)
	    f = a;
	  else
	    f = b;
	end
	```

## 3. Finite State Machine (FSM)
### 3.1 Moore vs Mealy
**Moore machine（摩尔机）**：$output=g(CS)$
输出只由当前状态决定。所以状态图里一般把输出写在 state circle 里，例如 `B/1`。
**Mealy machine（米利机）**：$output=g(CS,input)$
出由当前状态和当前输入共同决定。所以状态图里一般把输出写在 transition arc（转移边）上，例如 `input/output`。

### 3.2 三种 FSM 编码方式
1. Binary Encoding / Compact Encoding（二进制紧凑编码）
	这是最普通的编码方式。用尽可能少的 bit 表示所有状态。
	如果有 M 个状态，需要：$[log_2​M]$ 个 flip-flops。
2. One-hot Encoding（独热编码）
	One-hot encoding 是每个状态对应一个 bit，任何时候只有一个 bit 为 1。
3. Output Encoding / State-assigned-by-output（按输出特征编码）
	如果某些状态的输出刚好不同，可以把 state bits 设计得让输出直接等于某些 state bits，从而简化 output logic。
	例如一个 Moore FSM 有 4 个状态，每个状态输出 `y` 如下：
	```
	S0: y = 0  
	S1: y = 0  
	S2: y = 1  
	S3: y = 1
	```
	可以故意让最高位表示输出：
	```
	S0 = 00  
	S1 = 01  
	S2 = 10  
	S3 = 11
	```

## 4. Timing Rules
同步电路的基本模型是：
> FF/register output → combinational logic → FF/register input

时序设计目标：让所有状态只在 clock edge 更新，而 clock edge 之间留给组合逻辑完成计算
### 4.1 主输入/输出 Primary Input / Primary Output
在同步电路中，primary inputs 通常来自前一级 FF outputs，primary outputs 通常送到下一级 FF inputs。
### 4.2 Critical Path（关键路径）与 Short Path（短路径）
1. **Critical path**
	从某个 primary input 到某个 primary output 的最长延迟路径。它决定组合逻辑最终稳定需要多久。
	PI 到 PO 的最长延迟，即 **propagation delay（传播延迟）**
2. **Short Path**
	是 PI 到 PO 的最短延迟路径，即 **contamination delay（污染延迟）**
3. 特殊情况
	**Reconvergent Fanout（重汇聚扇出）**
	当一个信号扇出到多个门，之后又重收敛到同一个信号时，就叫 reconvergent fanout。此时每个点的 delay 不能简单用一个数表示，而往往变成一个 ⟨min,max⟩ 范围。
	**False Path（假路径）**
	false path 是物理上存在，但逻辑上不会影响输出的路径。比如某些 mux 的选择条件使得一条看起来很长的路径永远不会真正被激活。

### 4.3 Flip-Flop Timing
三个重要时序参数
- Setup Time $T_{setup}$
	clock edge 到来前，D input 必须提前稳定的最小时间。
- Hold Time $T_{hold}$
	clock edge 到来后，D input 必须继续保持稳定的最小时间。
- Clock-to-Q Time $T_{clk2Q}$
	clock edge 之后，FF output Q 还没有最终稳定的时间。细分为 $T_{ccq}, T_{pcq}$
输入必须在 clock edge 前 $T_{setup}$ 到达，并且在 clock edge 后保持 $T_{hold}$，才是一个对于该FF而言有效的输入。

### 4.4 时钟频率
1. 没有时钟偏移
	**Cycle Time Constraint 与最高时钟频率：**
	critical path 是整个系统中从任意 FF output 到任意 FF input 的最长组合路径；我们希望尽量减小 $T_{cyc}$​，因为更小周期意味着更高频率。
	$$\displaylines{T_{c l k2Q,m a x}+T_{c o m b,m a x}+T_{s e t u p}\leq T_{c y c}\\
	T_{c y c,m i n}=T_{c l k2Q,m a x}+T_{c o m b,m a x}+T_{s e t u p}\\
	f_{m a x}={\frac{1}{T_{c y c,m i n}}}
	}$$
	其中：
	- $T_{clk2Q,max}$：源 FF clock edge 后 Q 最晚稳定时间；
	- $T_{comb,max}$：组合逻辑最长路径，即 critical-path delay (Tpd)；
	- $T_{setup}$：目标 FF 在下一拍前需要的建立时间；
	- $T_{cyc}$​：clock period。
	**Hold Time Constraint :** 
	如果 FF1 和 FF2 之间几乎没有逻辑，新值可能非常快从 FF1 到 FF2，导致 FF2 的输入在 clock edge 后太快改变，违反 hold time。
	$$T_{c l k2Q,m i n}+T_{c o m b,m i n}\geq T_{h o l d}$$
	其中：
	- $T_{clk2Q,min}$：源 FF 输出最早开始变化的时间；
	- $T_{comb,min}​$：组合逻辑最短路径，也就是 contamination delay (Tcd)；
	- $T_{hold}​$：目标 FF 要求输入在 clock edge 后保持稳定的时间。

在去年的ECE2050中，这里的 $T_{clk2Q,max}$ 即为 $T_{pcq}$, $T_{clk2Q,min}$ 即为 $T_{ccq}$
![[Pasted image 20260504221054.png]]

2. 有时钟偏斜 Clock Skew
	如果源 FF 看到 clock 比目标 FF 晚，等效可用时间变少。$$T_{c l k2Q,m a x}+T_{p r o p a g a t i o n}+T_{s e t u p}\leq T_{c y c}-T_{s k e w(s r c-d e s t)}$$
	如果源 FF 看到 clock 更早，目标 FF 看到 clock 更晚，新数据可能更容易冲进目标 FF 的 hold window。$$T_{c l k2Q,m i n}+T_{c o n t a m i n a t i o n}\geq T_{h o l d}+T_{s k e w(a e s t-s r c)}$$

Big No-No: Logic on Clock，也就是不要随便门控时钟（在时钟线上加入门级电路逻辑）

# Section 2. 汇编 Assembly Language
## 1. 可编程机器
### 1.1. Stored-Program Computers
指令和和数据存储与同一mem中
基本运行逻辑：
- **Fetch**：从 memory 取 instruction
- **Decode**：解释 instruction 的含义
- **Execute**：执行 instruction
- **Update PC**：确定下一条 instruction 的地址
- 重复上述过程。
重要部件：
- **PC (Program Counter)**：保存“下一条/当前要执行的 instruction 的地址”
- **IR (Instruction Register)**：保存“当前正在执行的 instruction”
- **ALU (Arithmetic Logic Unit)**：执行算术和逻辑运算
- **Memory**：同时存 instruction 和 data
### 1.2. 指令集架构 (ISA, Instruction Set Architecture)
**ISA** 是程序员与可编程机器之间操作的规则说明。（作为interface）
规定
- 程序员可见状态（visible state）是什么
- 有哪些 instruction，以及每条 instruction 如何改变这些状态。

![[Pasted image 20260422202830.png]]

>E.g. P3070 ISA 的可见状态：
> **Memory**
     16-bit 地址
     每个地址存 16-bit word
     总空间为 64K words
   **8 个通用寄存器**
     R0 ~ R7，16-bit
   **2 个专用寄存器**
     PC: program counter，是当前/将要 fetch 的 instruction address。正常顺序执行时，处理器会根据指令长度更新 PC：short format 加 1，long format 加 2；branch / JSR / RTN 会改变 PC 到非顺序地址。
     SP: stack pointer
   **4 个条件码（Condition Codes, CC）**，存储ALU运算结果
     `Z`：zero，结果为0则 Z = 1
     `N`：negative，结果为负则 N = 1
     `C`：carry，对无符号数运算，结果最高位溢出/需要借位 C = 1
     `V`：overflow，对有符号数运算，溢出 V = 1

## 2. 汇编指令
ISA覆盖的instruction必须支持以下几类：
- 数据移动
- 数据运算
- 控制流
- 子程序，栈支持
### 2.1. P3070 基础指令
对例子P3070而言，分别如下：
1. Load Store Transfer
	- `LDA Rd, addr`：`Rd ← Mem[addr]`
	- `LDI Rd, imm`：`Rd ← imm`
	- `LDR Rd, Rs`：==`Rd ← Mem[Rs]`==
	区分 A: address, I: immediate value (Constant), R: register
	- `STA addr, Rs`：`Mem[addr] ← Rs`
	- `STR Rd, Rs`：`Mem[Rd] ← Rs`
	- `MOV Rd, Rs`：`Rd ← Rs`。
	注意: P3070 中 branch 依赖最近一次更新的 Condition Codes。
	Load 类指令如 LDA/LDI/LDR 的加载结果也会更新 CC。
	```asm
	LDA R0, PATRN  
	BRZ DONE  
	或：  
	LDA R2, $21  
	BRN GotOne
	```
	
2. Arithmetic / Logic: 不直接访问 memory，而是在寄存器上做运算，并更新 Condition Codes
	- `ADD`, `SUB`（都是前加/减后）, `NEG`, `INCR`, `DECR`
	- `AND`, `OR`, `NOT`
	- `LSHL/LSHR` 等 shift 指令：Logic shift left/right, 移出值作为进位C

3. Branch
	- `BRA addr`：无条件跳转（goto）
	- `BRN addr`：看 `N`，若最近一次更新 condition codes 的 ALU/compare/shift 结果为负值（N = 1），则下一条指令跳转到addr处，否则继续按顺序执行。
		if (N), PC <- addr, else PC++
	- `BRZ addr`：看 `Z`
	- `BRV addr`：看 `V`。

### 2.2. 指令格式与机器码
指令分为 opcode 和 operand 两部分
P3070 采用 **destination 在前，source 在后** 的书写规则
1. 短格式 Short Format - 1 word
	addrX opcode(10bit) ra(3bit) rb(3bit)
2. 长格式 Long Format - 2 word
	addrX opcode(10bit) ra(3bit) rb(3bit)
	addrX+1 address or data (Imm)


LDA Rd, addr
>addr X: opcode = 0000010000 ra = Rd = 111 rb = Rd = 111
>	word = 0000010000 111 111
>addr X+1: addr = hex 1234 = 0001001000110100
>hex 043F 1234

### 2.3. 寻址方式
1. Register mode: operand 直接在寄存器中，不需要额外地址字段
	e.g. `MOV Rd, Rs`、`INCR Rd`, 常为短格式
	
	**MOV Rd Rs**
	```
	opcode = 0011101000 ra = Rd = 100 rb = Rs = 101
	word = 0011101000 100 101 = 3A25
	```
	
	**INCR Rd**
	```
	opcode = 0001010000 ra = Rd = 011 rb = Rd = 011
	word = 0001010000 011 011 = 141B
	```
2. Absolute mode: operand 在 memory 中，而 地址本身直接写在 instruction 后面的第二个 word
	e.g. `LDA Rd, addr`、`STA addr, Rs`
	
	**LDA Rd, addr**
	```
	addr X: opcode = 0000010000 ra = Rd = 111 rb = Rd = 111
	word = 0000010000 111 111
	addr X+1: addr = hex 1234 = 0001001000110100
	hex 043F 1234
	```
3. Register Indirect mode: 寄存器里存放的是地址，真正的数据在该地址对应的 memory 位置 (pointer / address-of)
	e.g. `LDR Rd, Rs`、`STR Rd, Rs`
	
	**LDR Rd, Rs**
	```
	opcode = 0000100000 ra = Rd = 010 rb = Rs = 011
	word = 0000100000 010 011 = hex 0813
	```
4. Immediate mode: 操作数不是 memory 中的数据，而是 instruction 自己携带的常数
	e.g. `LDI Rd, imm`
	
	**LDI Rd, imm**
	```
	addr X: opcode=0000110000 ra = Rd = 111 rb = Rd = 111
	word = 0000110000 111 111
	addr X+1 addr = hex 1234 = 0001001000110100
	hex 0C3F 1234
	```
### 2.4. 会更改状态码的指令
1. `CMI Rd, imm` ; compare Rd with data imm, set CC
	e.g. `CMI R3, $abcd` : really calculates (R3 - $abcd) and sets the CCs
	Long Format
2. Shift Operations
	1. `LSHR Rd`、`LSHL Rs` ; logical shift right/left Rd by 1 bit
		e.g. `LSHR R2` ; logical shift right R2 by 1 bit; Insert a 0 in vacated bit; p ➙ Carry(C)
	
	2. `ASHR R2` ; arithmetic shift right R2 by 1 bit;
		Insert back the high-order bit (representing sign of 2s comp number) into the vacated bit; p ➙ Carry
		！没有`ASHL`！
	3.  `ROL R2` ; rotate R2 by 1 bit through Carry![[Pasted image 20260422222120.png]]
### 2.5. 伪操作
给 assembler 看，而不是处理器执行；因此不会被编译成可执行的指令
1. `.EQU` (equal)
	`ricky .EQU $56` Defines a constant $56 named "ricky"
	= \#define
2. `.DW` (direct write)
	用于定义变量初始值
3. `.ORG` (organize)
	`.ORG $a000` Stuff that follows should go in memory starting at address $a000
e.g.
```asm
	  .ORG $a000 (接下来的内容和从$a000开始放)
fred  .DW $00ff （在当前地址--$a000写入$00ff）
	  .DW $abcd 
ethel .DW $1234  （`ethel` 作为一个符号，代表的是“这块数据所在的位置”，也就是$a002）
```

## 3. 数据结构与子程序
### 3.1 数组 Array
![[Pasted image 20260505222532.png]]
数组在mem中表现为连续的一段地址，因此在每次读取数组内容时：
- 先从mem中读出索引 I，存入寄存器中（R3），`DLI`
- 在寄存器内进行地址索引的相加：将数组开头元素的地址读入寄存器（ADI R7，Y），`ADD`
- 使用`LDR`，从R7寄存器所指的地址中取出数值，存入寄存器R0。
- 将索引 I 读入R3，进行自增 `INCR` 后存回mem，`STA`
判断循环是否终止终止：
- `CMI` 将R3减去`imm`($5)，设置CC
- `BRN`，若N为1说明结果为负，I < 5；`BRZ`，若Z为1说明结果为0，I = 5：两种情况下都继续循环至`LOOP`代码块，否则跳入`NEXT`。

### 3.2 栈与子程序 Stack and Subprogram
#### 3.2.1 Program Counter, PC
用于存储当前正在执行的 instruction 的地址。一般情况下，处理器执行完一条指令后会自动更新 PC：
- 短格式指令 short format：`PC ← PC + 1`  
- 长格式指令 long format：`PC ← PC + 2`
改变PC变化：branch指令，如`BRA taget`，即为子程序调用。
在这个过程中，需要跳到 `subroutine`的同时，保存`return address`：存入stack。

#### 3.2.2 SP 和 Stack
栈作为LIFO的数据类型，后入先出。在本节课中，栈是一块特殊的 mem，而 SP（Stack Pointer） 是一个 16-bit register，保存当前 stack top，也就是最新压入栈的元素地址（指向最后一个填有数据的地址）。
本节课的栈从$FFFF开始（栈底）向低地址增长。
相关汇编指令：
```asm
PUSH Rs: 先 SP--，再写 Mem[SP]
	SP ← SP - 1  
	Mem[SP] ← Rs
POP ：先读 Mem[SP]，再 SP++
	Rd ← Mem[SP]  
	SP ← SP + 1
```
> 注意，`POP` 不会真的把 memory 里的旧值擦掉。它只是让 SP 移走。以后这块 memory 不再被认为属于当前 stack 内容。

初始化栈：
```asm
LDI R0 $0
LDSP R0
```
> 第一次`push`之后，SP -= 1，第一个元素被存入$FFFF。因此初始化时 SP 为 $0000

#### 3.2.3 子程序调用
`JSR addr`: Jump to SubRoutine (At \[addr])
该指令完成三件事
```
SP ← SP - 1  
Mem[SP] ← PC + 2  (Long format, 直接写入返回地址，保存至栈中)
PC ← addr  (将跳转地址写入PC，跳转)
```
把返回地址 push 到 stack，然后 PC 跳到 subroutine 开头。

`RTN`: Return from subroutine
该指令完成两件事
```
PC ← Mem[SP]  (将跳回地址从栈中写入PC，跳回)
SP ← SP + 1  (POP)
```
`RTN` 从 stack top 拿出 return address，放回 PC。这样程序就能回到调用点后面的那一条指令继续执行。
![[Pasted image 20260505232854.png]]

#### 3.2.4 保存寄存器
防止子程序运行中使用了寄存器进行运算，则有可能覆盖主程序在其中写入的数据（clobber registers）。因此，需要对寄存器数据进行保存。
区别于caller saved，本节课使用callee saved。
```asm
JSR FRED
;work with R1 R2

FRED:
    PUSH R1
    PUSH R2
    ; use R1, R2 freely
    POP R2
    POP R1
    RTN
```
> 子程序开始运行前，先将自己需要用到的寄存器数据`push`入栈，运行结束后，逆序`pop`出栈，存回寄存器中。

### 3.2.5 参数传递
类似，我们依然使用stack进行参数传递。不过不同的是，caller负责将参数压栈 (Left-to-right order)。
```asm
PUSH A   ; Pass Paras
PUSH B
JSR FRED
POP B    ; Remove Paras
POP A

FRED:
	get A from stack
	get B from stack
	put X into R7 (返回值)
```
总结：P3070的Stack discipline

#### 3.2.6 栈帧 Stack Frame
Stack frame = 一次 subroutine call 在 stack 上占用的那一片区域。
![[Pasted image 20260506002432.png]]
它里面通常包括：
- return address  
- parameters  
- saved registers  
- local variables（子程序用到的局部变量）

**Parameter Access**
`LDSF Rd, $num`
意为`Rd ← Mem[SP + num]`，也就是从当前 stack frame 的某个 offset 处读数据。
`STSF Rs, $num`
意为`Mem[SP + num] ← Rs`，也就是把寄存器里的值写到当前 stack frame 的某个 offset 处。
例如：
```asm
PUSH R5   ; Pass Paras
PUSH R6
JSR FRED
POP R6    ; Remove Paras
POP R5

FRED PUSH R0 
	 LDSF R7, $2 ;R7 ← B 
	 LDSF R0, $3 ;R0 ← A 
	 ADD  R7, R0 ;R7 ← A+B 
	 POP  R0 
	 RTN
```
>  注意，FRED 不会永久改变这次调用的 stack frame；也就是说，FRED 在返回之前会把自己造成的 stack 变化撤销掉。

但如果参数只是用来传递，返回后并不想恢复到 R5、R6，其实没必要真的 `POP` 到寄存器。只要把 SP 恢复即可：
`ADDSP $2`：`SP ← SP + 2`，可以一次性把多个参数从 stack frame 里“丢掉”。

**Local Variables**
```C
int revenant(int a, int b) {
    int c, d;
    ...
}
```
其中`c, d` 是 local variables（局部变量）。它们只在函数执行期间存在。函数返回后，这个 stack frame 被释放，局部变量也就不存在了。
局部变量存储在stack frame中，使用`LDSF` / `STSF`访问。

## 4. Assembly Timing/IO
### 4.1 程序计时 Program Timing
每条 P3070 instruction 有固定或近似固定的 clock cycle 数，程序总时间就是动态执行过的指令周期数之和。数据：查表。
static execution：timing does not depend on data
dynamic execution：timing dependson data（循环体等）

**Programs are made up of basic blocks**
Basic block（基本块） 是一段 straight-line code，中间没有跳转，通常以 branch 结束。程序一旦进入基本块，开始顺序执行，中途不进行跳转。
$$T_p=\sum_bT_b \cdot Frequency_b$$
> 程序总时间 = 每个 basic block 的时间 × 这个 block 被执行的次数，求和。

### 4.2 内存映射 I/O Memory-Mapped I/O
外设的控制寄存器、状态寄存器、数据缓冲区，被放在特殊的 memory address 上。程序访问这些地址时，并不是真的在访问普通 RAM，而是在访问外设寄存器。所以程序可以使用`LDA，LTA`等指令访问I/O。
设备寄存器被分为两类：
- control and status — a way to control and sense what the device is doing
- data buffers — where to read or write data
### 4.2.1 Exp. 数模转化器 ADC
![[Pasted image 20260506124459.png]]
三个关键寄存器：
- $10：control register  
- $20：input data buffer  
- $21：status register
工作流程：
- 程序往 `$10` 的 bit 0 写 1，使能设备。
- ADC 每 10 microseconds 产生一个 sample，放到 `$20`。
- 同时 ADC 会把 `$21` bit 15 的 data-ready bit 置 1。
	（此时$20的符号位为1，值为负，读入寄存器后可以直接更新状态码N为1。）
- 程序读取 `$20` 后，ADC 清除 data-ready。

### 4.2.2 Interrupt
Busy-Waiting 问题：CPU时钟速度远大于ADC数据产生速度，所以大部分情况下CPU都在等待（polling，轮询），处理器利用率低。
Interrupt（中断） 的思想是：CPU 不要一直等外设。CPU 去执行别的程序，等 ADC 准备好数据时，ADC 发出 interrupt request，CPU 在两条 instruction **之间**检查到这个请求，然后临时跳去执行 **ISR**（interrupt service routine，中断服务程序）。
> ISR 是 external event 发生时被调用的程序
> 中断和普通 `JSR` 很像，但更强制。普通 `JSR` 是程序自己写的函数调用；interrupt 是外设事件触发的“临时函数调用”。

interrupt 发生时，CPU 自动进入 interrupt state:
```
PUSH PC  
PUSH Condition Codes  
PC = Mem[$30]  
Go to Fetch
```
> `JSR`：主要保存 return address  
> Interrupt：保存 PC + Condition Codes

`RTI`(Return to Interrupt)
返回指令，完成：
```
POP stack to CC  
POP stack to PC
```

**中断向量表 Interrupt Vector Table**
CPU 收到 interrupt 后，不能凭空知道 ISR 地址。中断向量表用于解决这个问题。
在外设的映射内存地址上的一部分划定为 interrupt table，用于存储中断类型和 ISR 的映射表：
```
$30 : ADC_ISR 的起始地址  
$31 : Timer_ISR 的起始地址  
$32 : Keyboard_ISR 的起始地址
```
真正发生中断时，硬件会告诉 CPU：这次是哪一个 interrupt vector number。然后 CPU 用这个编号去查表，将 ISR 的地址存入 PC 中。
![[Pasted image 20260506132427.png]]

### 4.2.3 ISR
ISR 本质上也是一段程序，与子程序很像。但是区别在于：
- ISR 不是由 `JSR` 调用的，所以它一般没有 caller 通过 stack 传参数。因此，`ISR`需要`StISR`程序块进行 setup，使用全局变量（mem \[ \* ]) 来传递参数。
- ISR 会打断正在运行的程序，所以它必须小心保存自己会用到的 registers；与 Subroutine 类似，在开始时 `PUSH`，在最后反向 `POP`。
- ISR 结束时用 `RTI`，不是 `RTN`。
**Exp.**
`StISR`: 普通 subroutine。它的任务不是读 ADC，而是把之后 ISR 要用的信息先存起来，并且 enable device interrupt。调用：
`StISR(addrOfArray(YADDR), numOfSamples(NUM))`
所以 caller 会把两个参数放到 stack 上：
```
SP+0 : return address
SP+1 : numOfSamples
SP+2 : addrOfArray
```
```asm
StISR LDSF R7,$2  
STA YADDR,R7  ;把数组地址存到全局 memory variable `YADDR` 里
LDSF R7,$1  
STA NUM,R7  ;把 sample 数量存到 `NUM` 里
LDI R7,$3  
STA $10,R7  
RTN  
  
YADDR .DW $0  
NUM .DW $0  
ARRAY .DW ...
```
![[Pasted image 20260506134551.png]]

ISR 部分：
```asm
ISR    PUSH  R0
       PUSH  R1
       PUSH  R2  ;保存自己要用的 registers

S      LDA   R0,YADDR
       LDA   R1,NUM
       LDA   R2,$20  ;读 ADC 刚刚准备好的 sample
       STR   R0,R2  ;把 ADC sample 存到数组当前位置
       INCR  R0
       STA   YADDR,R0  ;数组指针往后移动一格，并存回 `YADDR`
       DECR  R1
       BEQ   DONE  ;如果减到 0，就说明任务完成，跳到 `DONE`
       STA   NUM,R1  ;如果没完成，把新的剩余数量存回 `NUM`，供下一次 ISR 使用

CLEAN  POP   R2
       POP   R1
       POP   R0
       RTI

DONE   STA   $10,R1
       BRA   CLEAN
```

## 2. FSM
### 2.1 微指令 Micro - Instruction
> 假设：IR 已经装好了当前 instruction，PC 也已经指向 opcode 后面的那个 word。

FSM 输出的控制信号包括[[# 1.4 总结：最后所有的控制点 control points|上图这些信号]]，它们组合起来就是一条 micro-instruction（微指令）。例如：
`ADD R3, R2` ，各项即为 “A+B” "RF" "RF" "RF" 0 1 1
`CMR R5, R2`，“A-B” "RF" "RF" "None" 0 1 1 （不会改变寄存器值，但能影响后面的 `BRZ/BRN/BRC/BRV`）

### 2.2 ASM 是一串 μ-instructions
1. `LDI R4, $1987`
此时，mem中存储的 opration：
```
地址 X:     LDI R4, imm 的 opcode word
地址 X+1:   $1987    <- PC (X+1)
地址 X+2:   next instruction
```
我们将这个指令拆成三个步骤：
- Step 1: `MAR ← PC`
	`F = A, SrcA = PC, dest = MAR`
- Step 2: `MDR ← Mem[MAR], PC ← PC + 1`  
	`F = A + 1, SrcA = PC, dest = PC, RE active`
- Step 3: `RF ← MDR`
	`F = A, SrcA = MDR, dest = RF`
每一行就是 FSM 的一个 state。

2. `STA $1987, R2`
datapath 设计里，memory write 需要单独一个 [[#^WriteMem|write cycle]]：先 load MAR，再 load MDR，再 assert WE，最后 data 在 cycle end 写入 memory。
![[Pasted image 20260506171607.png]]
- Step 1: `MAR ← PC`  
	从 PC 中读入 "$1987" 这个数据位于 mem 中的地址
	``F = A, SrcA = PC, dest = MAR``
- Step 2: `MDR ← Mem[MAR], PC ← PC + 1`
	这一步读出 second word "$1987"；同时 PC 加 1，跳过 second word，指向下一条 instruction。
	`F = A+1, SrcA = PC, dest = PC, RE active`
- Step 3: `MAR ← MDR`  
	memory write 要求地址 ("$1987") 必须在 MAR 里
	`F = A, SrcA = MDR, dest = MAR
- Step 4: `MDR ← RF` `
	地址准备好了，还要准备数据。将 R2 数据写入 MDR。
	`F = A, SrcA = RF, dest = MDR`
- Step 5: `WE` active
	assert write enable
	`dest = none, WE active`

注意：
- Machine cycle（机器周期）就是一个 clock period，即是 FSM 的一个 state。一个 machine cycle 里可以发生一组 register transfer。
- Instruction cycle（指令周期）是执行一整条汇编指令的全过程。它可能包含多个 machine cycles。

### 2.3 如何产生控制信号：FSM（Control Path）
这个 FSM 的输出就是 control points。它的每个 state 输出一条 micro-instruction。
执行任何 instruction，都需要：
Fetch instruction -> Decode opcode -> Execute corresponding operation
![[Pasted image 20260506194016.png]]

**Fetch**
Fetch 前：PC holds the address of the next instruction
Fetch 后：`IR ← Mem[PC]  PC ← PC + 1`
	fetch 后 PC 指向 memory 的下一个 word，可能是 second word（Long）或是下一条指令（Short）。
具体的微操作：（类似`LDI`)
```
MAR ← PC  
	F = A PC xx MAR 1 1 1
MDR ← Mem[MAR], PC ← PC+1  
	F = A + 1 PC xx PC 1 0 1
IR ← MDR
	F = A MDR xx IR 1 1 1
```
